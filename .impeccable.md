# Impeccable — mdblog design context

This file was generated by the **teach-impeccable** workflow. Update it when brand or audience intent changes.

## Design Context

### Users

- **Primary use**: A personal Hexo blog theme (`mdblog-hexo`): readers open posts written in Markdown, sometimes with **KaTeX** math, and browse by **category** from the top app bar.
- **Author context**: Config and copy skew toward a **bilingual** (Chinese + French) voice and an **intellectual / historical** motif (e.g. Galois banner and cite in `_config.yml`).
- **Job to be done**: Find and read long-form content quickly; navigate categories; optional dark/light comfort via system theme (`mdui-theme-auto` on `<html>`).

### Brand Personality

- **Stated product tone** (README): 简约、Markdown 风格、简洁 — **minimal, content-first, “markdown blog”** feel.
- **Surface personality** (from default banner copy): **youthful energy**, **progressive**, **courageous**, layered with **literary / scholarly** citation — not a cold corporate blog.
- **Implementation voice**: **Material Design 3** via **MDUI 2** (components, tokens, typography scales), plus **Material Symbols** / **Font Awesome** for brand icons — polished and familiar, not bespoke illustration-heavy.

### Aesthetic Direction

- **Visual system**: Rely on **MDUI semantic tokens** (`--mdui-color-*`, `--mdui-typescale-*`) and **MDUI prose** for article body; custom CSS stays **minimal** and documents itself as “only what MDUI cannot do.”
- **Layout**: Centered **`.mdui-container`** (max-width **1280px**, fluid side padding). **首页**：主内容单栏（`page-grid--single`），`post-index-grid` 列宽封顶（`minmax(..., 360px)`）+ `justify-content: center` 使卡片组在版心内居中。**文章页**：宽屏 **~840px+** 为 **主栏 3fr + 侧栏 1fr**，侧栏仅 **目录（TOC）**（`sticky-sidebar` + `--app-top-bar-offset`）；`toc: false` 时不渲染侧栏，主栏 **`page-grid--single`** 单栏。**分类列表页**：**`page-grid--single`**，主栏内 **`.category-index`** 全宽 **`notes-header`**（Display 级标题 + 可选导语：`theme.category_page.by_category[].class_name` / `class_description`，否则 `theme.category_page.lede` / `category_page_default_lede`）+ **`notes-grid`** 响应式网格，每篇为 **`mdui-card`**（`variant="outlined"`、`clickable`、`href`，`partial/category-note-card`）与分页。**顶栏右侧**：**`.topbar-profile-cluster`**：头像作 **`mdui-dropdown`** 触发器（`__('topbar_profile_open')`）；弹层内 **`mdui-card`** 含头像/简介 + **`关于`**（`mdui-button` 链至 `/about`），**无**内嵌社交列表；**社交**仅为顶栏 **`topbar-profile-social-bar`**（**≤600px 隐藏**）。顶栏 `padding-inline` 兼顾 **safe-area**。**无 FAB**。
- **Shape**: Default corner tuning **`--mdui-shape-corner-medium: 8px`** at `:root`.
- **Motion**: **Hero / banner** transitions exist; **`prefers-reduced-motion: reduce`** is respected in custom CSS — keep new motion consistent with that pattern.
- **Theme**: **Light and dark** follow **system** (`mdui-theme-auto`); new UI must work in both.
- **Hero**: Full-viewport-style banner (`~95vh` hero on index), image via **`--banner-bg-image`**, overlays and bilingual title stacks are part of the brand surface.

### Surfaces & `mdui-card`（设计语言）

- **`elevated`**：默认内容面 — 首页文章卡片、文章正文卡、**顶栏名片弹层**。分类索引页为 **`outlined` 可点卡片**（`mdui-card` + `href`），与首页 `elevated` 区分层次。与页面背景拉开层次，不抢正文。
- **`filled`**：保留给将来需要**实心强调面**的场景；当前主题布局中分类页已不使用独立 `filled` 信息卡。
- **内页顶栏横幅**：使用 `banner-inner*` + `banner-hero-title--inner`，**不**再使用 MDUI 1 的 `mdui-card-primary` / `mdui-float-*`。
- **主题色**：浏览器 `theme-color` 由 `_config.yml` 的 `theme_color` 提供，默认与 MD3 主色倾向一致；页面内仍以 `--mdui-color-*` 为准。

### 横幅标题布局 JSON + JS（列宽 / 换行 / 对齐）

- **文案数据**仍在 `_config.yml` 的 `banner.title_inner_columns` / `subtitle_block`（与 i18n 无关的作者内容）。
- **版式策略**由 **[source/js/banner-title-layout.json](source/js/banner-title-layout.json)** 描述，**[source/js/banner-title-layout.js](source/js/banner-title-layout.js)** 在文章页横幅（[layout/partial/banner.ejs](layout/partial/banner.ejs)）通过 `fetch` 加载 JSON，在 `[data-banner-layout-root]` 内匹配 `entries[].selector`，设置 `data-banner-align`、`--banner-layout-gap`，并视宽度挂上 **`is-banner-title-wrap`** / **`is-banner-title-stack`**。
- **站点覆盖**：在站点 `source/js/banner-title-layout.json` 放置同名文件即可覆盖主题默认（Hexo 生成后同 URL）。
- **字段（v1）**：顶层 `version`；`entries[]` 每项含 `selector`、`align`（`start` | `center` | `end`）、`wrapMode`（`nowrap` | `wrap` | `responsive`）、`columnGap`（如 `0.5rem`）、`childSelector`（测宽用，标题默认 `.banner-hero-title__col`，副标题 `:scope > *`）、`responsive.overflowRatio`（总宽/容器宽阈值）、`responsive.stackMaxWidth`（容器宽度 ≤ 该 px 时纵向堆叠）。
- **降级**：`fetch` 失败或无 JS 时保持纯 CSS 默认（内页标题区右对齐、`nowrap`）；若站点 CSP 禁止请求 JSON，需改为构建时内联配置（未实现）。

### CSS 模块化

- 入口仍为 **`source/css/custom.css`**（单 `<link>`），通过 **`@import`** 串联同目录子文件：`tokens.css`、`global-layout.css`、`topbar.css`、`hero.css`、`motion-scroll-reveal.css`、`lists-cards.css`、`categories-postinfo.css`、`banner-page.css`、`sidebar-footer.css`、`article-prose.css`。（**`sidebar-footer.css`** 中的 **`.sidebar-*`** 现主要服务于顶栏 dropdown 内名片；**`layout/partial/topbar-profile.ejs`** 为名片唯一 markup 来源。）
- 交互灰度优先用 **`tokens.css`** 中 `--app-*` 变量，避免散落魔法 `color-mix` 百分比。

### Design Principles

1. **MDUI first**: Prefer MDUI components, utility patterns, and CSS variables before new custom rules; add page-specific rules in the **appropriate `source/css/*.css` slice** (or `tokens.css` for shared hovers), aggregated by `custom.css`.
2. **Content readability**: Typography and spacing should serve **long reading**; link breaking, code blocks, figures, TOC density, and image max-height are already tuned — preserve or improve readability, not decoration for its own sake.
3. **Responsive and touch-friendly**: Top bar category nav scrolls horizontally on small screens; avoid layouts that break **min-width: 0** grid children or overflow without scroll affordances. 名片 **hover** 在触摸设备上不可靠 — 依赖 **`click` / `focus`** 打开 dropdown；`close-delay` 略大以便移入弹层。窄屏隐藏 **`topbar-profile-social-bar`** 时，当前无备用社交入口（名片内已不列链接）。
4. **Token-driven color and type**: Use **`rgb(var(--mdui-color-…))`** and **`color-mix`** for hovers/overlays to stay on-palette in light and dark.
5. **Respect user motion preferences**: Mirror existing **`prefers-reduced-motion`** handling when adding transforms, parallax, or large transitions.
6. **i18n-aware chrome**: User-facing strings in layouts use **`__('…')`** where applicable; banner block copy in theme config is **author-owned** and may stay out of i18n by design.
